# Curriculum authoring guide

Every lesson is a single JSON file under `/curriculum/` validated by `schema.json`.
The app consumes them from `assets/curriculum/` (synced at build time by `scripts/sync-curriculum.sh`).

## Tracks (in canonical order)

1. `fundamentals`  — atoms, lists, evaluation, `quote`, basic forms.
2. `functions`     — `defun`, `let`, recursion, higher-order, lambda, closures.
3. `data`          — strings, characters, numbers (incl. ratios/bignums/complex), arrays, hash-tables, structs.
4. `controlflow`   — `if`, `cond`, `case`, `when/unless`, `do`, `loop` macro, `tagbody`/`go`, `block`/`return-from`.
5. `macros`        — quasi-quote, `defmacro`, hygiene patterns, compiler macros, reader macros.
6. `clos`          — `defclass`, generic functions, `defmethod`, multiple dispatch, MOP teaser.
7. `conditions`    — `error`, `signal`, `handler-case`, `handler-bind`, `restart-case`.
8. `packages`      — `defpackage`, `in-package`, exporting, symbol resolution.
9. `system`        — ASDF: defining systems, dependencies, components.
10. `libraries`    — Quicklisp tour: alexandria, serapeum, cl-ppcre, drakma, hunchentoot, bordeaux-threads, cl-json, local-time, ironclad, cffi.

## Section kinds

* `prose` — Markdown explanation.
* `example` — runnable snippet with expected output (for "Run" button).
* `exercise` — prompt + starter + automated tests run inside the sandbox.

## Quality bar

* Every concept must have at least one runnable example.
* Examples must be self-contained (no hidden state from earlier lessons unless explicitly noted).
* Exercises must have at least one test that passes the reference solution.
* Lesson should fit a 5–15 minute slot.

## Naming

`NNN-kebab-id.json` where NNN is a global ordering. The `index.json` manifest
is generated by `scripts/build-curriculum-index.sh` (or hand-edited if you prefer).
